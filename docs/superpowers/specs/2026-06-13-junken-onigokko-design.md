# じゃんけん鬼ごっこ シミュレーション 設計書

日付: 2026-06-13
ステータス: 承認待ち

## 概要

じゃんけん鬼ごっこ(グー・チョキ・パーの3グループが三すくみで追いかけ合う遊び)を
ブラウザ上でシミュレーションし、リアルタイムに可視化する。

### ゲームルール

1. グー・チョキ・パーの3グループに分かれる。各グループ N 人(デフォルト10人)
2. グーはチョキを、チョキはパーを、パーはグーを捕まえられる(三すくみ)
3. 各メンバーは「獲物を追いかけ、脅威から逃げる」
4. 捕まったメンバーは盤面から除外される
5. 制限時間(デフォルト60秒)経過後、残存人数が最も多いグループの勝ち

## 決定事項

| 項目 | 決定 |
|---|---|
| 実行環境 | Webブラウザ。素のJavaScript + Canvas 2D。外部依存ゼロ、`index.html` をダブルクリックで起動 |
| メンバーの見た目 | 手の絵文字(✊ ✌️ ✋)+ グループ色の光彩(グロー) |
| グループ色 | グー = 赤 `#e74c3c`、チョキ = 緑 `#2ecc71`、パー = 青 `#3498db` |
| 画面レイアウト | 左サイドバー型(コントロール・残数・タイマーを左に常時表示、右に盤面) |
| 終了条件 | ①設定時間の経過、②膠着状態(残存グループが1つ以下) |
| 速度設定 | 全体共通(平均・標準偏差を各1つ設定し、全参加者が同じ正規分布から速度を引く) |

## パラメータ(サイドバーのスライダーで設定)

| パラメータ | 範囲 | デフォルト |
|---|---|---|
| 各グループの人数 N | 1〜100 | 10 |
| シミュレーション時間 | 10〜300 秒 | 60 秒 |
| 平均速度 | 10〜300 px/秒 | 80 px/秒 |
| 速度の標準偏差 | 0〜100 px/秒 | 15 px/秒 |

スライダー値の変更は次回スタート時に反映される(実行中の変更は反映しない)。

## シミュレーションモデル

### 盤面

- 論理座標系 800×600 の長方形に固定。CSS で表示スケールするため環境に依存しない
- 壁あり。エージェントの座標は盤面内にクランプされる

### エージェント

- 状態: 位置 (x, y)、速度スカラー、グループ、生存フラグ
- 個人の速度は正規分布(Box-Muller 法)から生成。最低速度 5 px/秒 でクランプ(負値防止)

### 初期配置

3グループを盤面の3つのコーナーエリアに分けてランダム配置する
(開始直後の即捕獲を防ぐ)。

### 行動ロジック(毎フレーム)

1. **獲物** = 自分が捕まえられるグループのうち最も近い生存者
2. **脅威** = 自分を捕まえられるグループのうち最も近い生存者
3. 移動方向 = 追跡ベクトルと逃走ベクトルの重み付き合成:
   - 追跡: `normalize(獲物の位置 − 自分の位置) × (1 / 獲物までの距離)`
   - 逃走: `normalize(自分の位置 − 脅威の位置) × (1.5 / 脅威までの距離)`
   - 合成結果を正規化し、個人の速度スカラーを掛けて移動する
   - 係数 1.5 により逃走をやや優先する(生存重視)。距離の逆数による重み付けで、
     近い相手ほど行動に強く影響し、切り替え式より滑らかな動きになる
4. 獲物が全滅していたら逃走のみ。脅威も獲物もいなければその場で停止

### 捕獲判定

- 2人の距離が捕獲半径 16 px 未満、かつ捕食関係にあれば捕まった側を除外
- 同一フレーム内では全捕獲ペアを収集してから一括除外する。
  「AがBを捕まえる瞬間にCもAを捕まえる」場合は両方とも有効(同時とみなす)。
  判定順序による不公平をなくすため

### 時間進行

- `requestAnimationFrame` + デルタタイム(実時間ベース)
- dt は上限 100ms にクランプ(タブ非アクティブからの復帰でワープしない)

### 終了と勝敗

- 終了条件: 設定時間の経過、または残存グループが1つ以下になったとき
- 勝者: 残存人数が最多のグループ。複数グループが同数の場合は引き分け
- 終了時は盤面上にオーバーレイで勝者(グループ名・色・最終人数)を表示

## UI仕様

```
┌──────────┬────────────────────────┐
│ 人数 N      ──○──   │                        │
│ 時間        ──○──   │                        │
│ 平均速度    ──○──   │       盤面 (Canvas)     │
│ 標準偏差    ──○──   │     ✊ ✌️ ✋ が動き回る   │
│ [スタート] [リセット] │                        │
│                      │                        │
│ ⏱ 残り 42.3 秒       │                        │
│ ✊ グー   7           │                        │
│ ✌️ チョキ 4           │                        │
│ ✋ パー   9           │                        │
└──────────┴────────────────────────┘
```

- 残数・タイマーは毎フレーム更新(リアルタイム)
- スタート: 現在のスライダー値で新規シミュレーションを開始
- リセット: 実行中でも停止して初期状態(配置のみ表示)に戻す
- 捕まったメンバーは即座に盤面から消える

## ファイル構成とコンポーネント

```
junken_onigokko/
├── index.html   … 画面(サイドバー+Canvas)、CSS、UI制御・描画のJS
├── sim.js       … シミュレーションロジック(描画と完全分離、Canvas非依存)
├── test.js      … sim.js のテスト(node test.js で実行、Node標準のassertのみ)
└── docs/superpowers/specs/ … 本設計書
```

`sim.js` は `<script src>` で読み込む普通のJS(非モジュール)。
末尾の `if (typeof module !== 'undefined') module.exports = ...` で Node からも読める。

| コンポーネント | 場所 | 役割 |
|---|---|---|
| `randNormal(mean, sd)` | sim.js | Box-Muller 法による正規分布乱数 |
| `Agent` | sim.js | 位置・速度・グループを保持。獲物/脅威の探索と移動方向の計算 |
| `Simulation` | sim.js | エージェント全体の管理。`tick(dt)` で移動→捕獲判定→終了判定 |
| `Renderer` | index.html | Canvas に絵文字+色グローで描画 |
| `UIController` | index.html | スライダー読み取り、スタート/リセット、残数・タイマー表示、rAF ループ |

### データフロー

```
スライダー値 → Simulation 生成 → 毎フレーム: tick(dt) → 状態
   → Canvas 描画 + 残数/タイマー DOM 更新
   → 終了検知 → 勝者オーバーレイ表示
```

## エラー処理・エッジケース

| ケース | 対処 |
|---|---|
| 速度の乱数が負になる | 最低速度 5 px/秒 でクランプ |
| 標準偏差 = 0 | 全員が平均速度ちょうどになる(正常動作) |
| 不正なパラメータ入力 | スライダー使用のため範囲外は入力不可 |
| 同一フレームで複数の捕獲が同時成立 | 全捕獲ペアを収集してから一括除外(すべて有効) |
| タブ非アクティブからの復帰 | dt を上限 100ms にクランプ |
| 画面サイズの違い | 論理座標 800×600 固定 + CSS スケール |
| 勝者が同数 | 「引き分け」と表示 |

## テスト(`node test.js`)

- **randNormal**: 大量サンプルの平均・標準偏差が指定値に収束すること
- **Agent**:
  - 最近接の獲物/脅威を正しく選ぶこと
  - 追跡・逃走の合成方向が正しいこと(獲物のみ→獲物方向、脅威のみ→反対方向)
- **Simulation**:
  - 捕獲半径内+捕食関係 → 除外される
  - 捕食関係がなければ除外されない
  - 同時捕獲が両方適用されること
  - 終了条件(時間切れ・膠着)が正しく発火すること
  - 勝者判定(最多残存・同数引き分け)
  - エージェントが盤面の外に出ないこと
