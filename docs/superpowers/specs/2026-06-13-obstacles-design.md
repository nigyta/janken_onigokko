# じゃんけん鬼ごっこ 障害物機能 設計書

日付: 2026-06-13
ステータス: 承認待ち
前提: `docs/superpowers/specs/2026-06-13-junken-onigokko-design.md`(実装済み)への機能追加

## 概要

盤面にランダムな長方形の障害物を配置する。

1. 障害物の中には入れない(進入不可)
2. 2人の間に障害物があると互いを認識できない(追跡・逃走の対象にならない)
3. 捕獲も遮られる(捕獲半径内でも壁越しには捕まらない)
4. 障害物の数はスライダーで設定できる

## 決定事項

| 項目 | 決定 |
|---|---|
| 視線判定 | 線分×矩形の交差判定(Liang-Barsky法)。枝刈り: 現在の最良候補より近い相手のみLOS判定 |
| 衝突解決 | 軸分離スライド(x のみ → y のみの順に試し、両方ダメなら停止) |
| 配置 | 完全ランダム(重なり許容)。エージェントは障害物外になるまで再抽選+「進入のみ禁止」ルールで自己回復 |
| 壁越し捕獲 | 不成立(視線が必要) |
| 経路探索 | しない。見えない相手は存在しないものとして扱い、誰も見えなければその場で停止 |

## データモデルとパラメータ

- 障害物はプレーンオブジェクト `{x, y, w, h}`(クラス不要)。`Simulation.obstacles` 配列に保持
- 新定数: `OBSTACLE_MIN_SIZE = 40`、`OBSTACLE_MAX_SIZE = 160`(px。各辺はこの範囲で一様ランダム)
- `Simulation` の params に `obstacleCount` を追加: `{ n, duration, meanSpeed, speedSd, obstacleCount }`
- 生成順: コンストラクタで障害物を先に生成 → エージェントを配置
  - 障害物は盤面内に完全に収まる(`x ∈ [0, FIELD_WIDTH − w]`、y も同様)。乱数は `this.rand`
  - エージェントの初期位置は担当エリア内で「障害物の外」になるまで再抽選(上限100回)。
    それでも見つからない場合は**盤面全体**から再抽選(さらに100回)— 障害物の総面積は盤面より必ず小さいため実質確実に成功する。
    最終フォールバックは最後の候補をそのまま採用(進入のみ禁止ルールにより脱出は可能)
- 障害物同士の重なりは許容(結合した形として機能するだけ)
- UIスライダー: 0〜15、デフォルト5。0 なら従来どおり障害物なし

## 視線判定(LOS)

- 新関数(sim.js、Canvas非依存):
  - `pointInRect(x, y, rect)`: 点が矩形内(境界含む)か
  - `segmentIntersectsRect(x1, y1, x2, y2, rect)`: 線分が矩形と交差するか(Liang-Barsky法。端点が矩形内の場合も交差とみなす)
  - `canSee(a, b, obstacles)`: a–b 間の線分がどの障害物とも交差しなければ true
- `Agent.nearestOf(agents, group, obstacles = [])`: 「見える最近接の生存者」を返す
  - 枝刈り: 距離が現在の最良候補より近い場合のみLOS判定を行う(典型的には1〜2回で済む)
- `Agent.direction(agents, obstacles = [])`: 見える獲物・脅威のみが対象。誰も見えなければ `null`(停止)
- 捕獲判定: 捕獲半径内 かつ `canSee(hunter, prey, obstacles)` のペアのみ成立
- `obstacles = []` のデフォルト引数により、障害物なしでは完全に従来挙動(既存テストは変更なしで通る)

## 衝突解決(進入のみ禁止+軸分離スライド)

毎フレームの移動処理:

1. 移動候補 `(nx, ny)` を計算(従来どおり方向×速度×dt、盤面端クランプ)
2. `(nx, ny)` がブロックされなければ採用
3. ブロックされたら `(nx, 現y)` → `(現x, ny)` の順に試す(壁に沿って滑る)
4. すべてブロックなら停止(現位置のまま)

「ブロックされる」の定義: **現在位置を含まない障害物に新位置が入ること**。
すでに障害物内にいる場合、その障害物については判定を免除する(=進入のみ禁止、脱出は常に許可)。
これにより、万一障害物内に配置されても自力で脱出でき自己回復する。

## UI・描画

- サイドバーにスライダー追加: 「障害物の数: <span>5</span>」(`min=0 max=15 value=5`)
- `readParams()` に `obstacleCount` を追加
- `draw()`: エージェントより先に障害物を描画。塗り `#2c2c54` + 枠線 `#44446e`(暗色で盤面に馴染ませる)
- 障害物はスタート/リセットのたびに再生成(他パラメータ同様「次回スタート時に反映」)

## エラー処理・エッジケース

| ケース | 対処 |
|---|---|
| 障害物がスポーンエリアを大きく覆う | 担当エリアで100回 → 盤面全体で100回の2段階再抽選。障害物内に配置されたまま始まる事態は実質起きない(万一の場合も進入のみ禁止ルールで脱出可能だが、障害物内では視線が通らず停止し続ける点に注意) |
| 障害物の中にいる状態での移動 | その障害物の判定は免除(脱出可)。他の障害物への進入は不可 |
| 全員視線が通らない | 各自停止。終了条件(時間切れ/膠着)は従来どおり機能する |
| obstacleCount = 0 | 障害物なし。従来挙動と完全一致 |
| 細い隙間(障害物の重なりの境界) | 特別扱いしない。通れなければスライド/停止するだけ |

## テスト(node:test に追加。既存25件は変更なしで通る)

- 障害物生成: 個数どおり / 盤面内に収まる / 各辺が 40〜160 の範囲
- 初期配置: 全エージェントが障害物の外
- `segmentIntersectsRect`: 横断する / かすらない / 端点が内部 / 矩形の脇を平行に通る
- LOS付き `nearestOf`: 最近接が壁の向こう → 見える次点を選ぶ / 全員見えない → null
- `direction`: 獲物も脅威も見えなければ null
- 移動: 障害物に進入しない / 壁に沿ってスライドする / 内部に置かれた場合は脱出できる
- 捕獲: 半径内でも壁が間にあれば不成立
- `obstacleCount: 0`: 障害物なしの従来挙動
