# じゃんけん鬼ごっこ ランダム脱出 設計書

日付: 2026-06-13
ステータス: 承認待ち
前提: `docs/superpowers/specs/2026-06-13-obstacles-design.md`(実装済み)への挙動追加

## 概要

このフレームで結局1ピクセルも動けなかったエージェントを、ランダムな方向に動かして膠着を脱出させる。

対象となる「動けなかった」状態:
1. ターゲット(獲物/脅威)を見て移動方向はあるが、3通りの軸分離スライド(そのまま・x のみ・y のみ)が全部障害物でブロックされた(障害物の角に挟まれた)
2. 盤面の角に押し付けられ、方向はあるが位置が端でクランプされ動けない
3. 相手が誰も見えず移動方向が無い(従来は停止)

3つすべてを「位置が変わらなかった」という一様な条件で拾う。

## 決定事項

| 項目 | 決定 |
|---|---|
| ランダム移動の方式 | 持続するランダム方位(wanderAngle)。詰まったらその向きへ進め、ダメなら引き直す(最大8回) |
| 発動条件 | ターゲット追従処理の後、位置が1ピクセルも変わっていない場合 |
| 適用範囲 | 上記1〜3すべて(相手が見えず停止していたエージェントも含む) |
| 乱数 | `this.rand`(Simulation 注入)を使用しテストで決定的に検証可能にする |
| 速度0の扱い | 移動量0なので自然に動かない(特別扱い不要) |

## 詳細

### 新規の状態・定数

- `Agent` に `wanderAngle`(初期 `null`)を追加。さまよう向きを保持する
- 定数 `WANDER_TRIES = 8`(ランダム方向の最大試行回数)

### `Simulation.tick` 移動ループの変更

現在の各エージェントの移動処理:

```js
const dir = agent.direction(this.agents, this.obstacles);
if (!dir) continue;
const nx = ...; const ny = ...;
// 軸分離スライド: そのまま → x → y、全部ダメなら停止
if (!this.blockedMove(agent, nx, ny)) { ... }
else if (...) { ... }
else if (...) { ... }
```

を以下の構造に変更する:

1. 移動前の位置 `ox, oy` を記録
2. `dir` があれば従来どおり軸分離スライドを試す(`dir` が null ならスキップ)
3. 処理後、`agent.x === ox && agent.y === oy`(=1ピクセルも動いていない)なら `this.wanderMove(agent, dt)` を呼ぶ

`continue` でループを抜ける形をやめ、方向が無くてもステップ3に到達できるようにする。

### `wanderMove(agent, dt)`(新メソッド)

- 最大 `WANDER_TRIES` 回繰り返す:
  - `agent.wanderAngle` が `null` なら `this.rand() * 2π` で初期化
  - その角度方向へ `速度 × dt` 進めた候補 `(nx, ny)` を盤面内にクランプして計算
  - 候補が現在位置と異なり、かつ `blockedMove` でブロックされなければ移動して終了
  - そうでなければ `agent.wanderAngle` を `this.rand() * 2π` で引き直して次の試行へ
- 全試行が失敗(完全に囲まれている)ならその場に留まる

持続する向き(`wanderAngle`)を最初に試すため、開けた場所では同じ方向に滑らかに進み、障害物や盤面端でブロックされたときだけ向きを変える。

## エッジケース

| ケース | 挙動 |
|---|---|
| 速度0 | 候補位置が現在位置と同じ → 移動せず留まる |
| 完全に囲まれている(障害物内など) | 8方向すべてブロック → その場に留まる(従来と同じ) |
| 盤面の角でターゲットに引かれている | クランプで位置不変 → wanderMove が発動し脱出を試みる |
| 相手が見えない(方向なし) | wanderMove が発動し探索的に動く(従来は停止) |

## 副次効果(意図的)

相手が見えないエージェントが探索的に動き回るため、全員が壁の裏に隠れて膠着し時間切れになる状況が起きにくくなる。これは「停止して膠着」より自然な挙動。

## テスト(node:test に追加。既存36件は維持)

- 角で詰まっていたエージェント(障害物に挟まれ位置不変)が、注入した乱数で決まる方向へ移動し、障害物の外に出る
- 相手が誰も見えないエージェント(速度>0、単独)が、停止せずランダムに動く
- 速度0のエージェントは動かない(既存挙動の回帰防止)
- 盤面の角でターゲットに引かれて動けないエージェントが wanderMove で動く
- 既存36テストが全部pass(特に速度0を使う捕獲・終了テストに影響がないこと)

## スコープ外(YAGNI)

- ランダム方位の定期的な変更(壁にぶつかるまで直進で十分)
- 経路探索や壁沿いの賢い回り込み
- wander 速度の個別調整(通常速度を使う)
- UI・描画の変更(挙動のみの変更)
