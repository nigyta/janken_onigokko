# じゃんけん鬼ごっこ 人数推移グラフ機能 設計書

日付: 2026-06-13
ステータス: 承認待ち
前提: `docs/superpowers/specs/2026-06-13-junken-onigokko-design.md`(実装済み)への機能追加

## 概要

グー・チョキ・パー各グループの生存人数の時間推移を、折れ線グラフでリアルタイム表示する。

1. シミュレーション進行中に、各グループの人数推移を折れ線で描き続ける
2. 終了後もグラフはそのまま残る(全期間の推移が見られる)
3. グラフはメイン盤面の下に配置する(横長)
4. 自前の Canvas 描画で実装する(外部ライブラリ・ビルド不要、オフライン動作維持)

## 決定事項

| 項目 | 決定 |
|---|---|
| 表示タイミング | リアルタイム(tick ごとに更新)。終了後もそのまま残す |
| 履歴の保持場所 | `Simulation`(レンダラではなくロジック側。既存の TDD パターンに沿わせテスト可能にする) |
| サンプリング間隔 | シミュレーション時間で 0.5 秒ごと。さらに `finished` の瞬間に最終状態を必ず1点記録 |
| 配置 | メイン canvas の下に新しいグラフ canvas を縦積み(`#board` を縦並びに変更) |
| 描画方法 | 自前 Canvas 2D 描画(Chart.js 等は使わない) |
| y 軸スケール | 0〜初期人数 `n`(人数は減る一方なので固定スケールで安定) |
| x 軸スケール | 0〜`duration` 秒 |
| 色 | 既存の `COLOR` を再利用(グー赤 `#e74c3c`・チョキ緑 `#2ecc71`・パー青 `#3498db`) |

## データモデル(sim.js)

`Simulation` に履歴とサンプリング用の時刻アキュムレータを追加する。

- 新フィールド:
  - `this.history`: サンプルの配列。各要素は `{ t, gu, choki, pa }`(`t` は経過秒、`gu/choki/pa` は生存人数)
  - `this.elapsed`: 経過時間(秒)。`duration - timeLeft` でも導出できるが、サンプリング判定を明示的にするため保持
  - `this.nextSampleAt`: 次にサンプルを取る経過時刻(秒)。初期値 `SAMPLE_INTERVAL`
- 新定数: `SAMPLE_INTERVAL = 0.5`(秒)
- コンストラクタ末尾で開始時点を記録: `this.history = [{ t: 0, gu: n, choki: n, pa: n }]`
  - 注: `n` は `params.n`。全グループ同数で開始する
- `pushSample()`(内部ヘルパ): 現在の `elapsed` と `aliveCounts()` から1点を `history` に追加

## サンプリングロジック(tick 内)

`tick(dt)` の処理順は従来どおり(移動 → 捕獲判定 → 終了判定)。これに履歴記録を加える。

1. 冒頭の `dt = Math.min(dt, MAX_DT)` の後、`this.elapsed += dt`(`timeLeft` 減算と対になる)
2. 捕獲判定・終了判定まで済ませた**後**に履歴を記録する(その tick で捕まった結果を反映するため):
   - `finished` になったら、間隔に関係なく最終状態を1点記録して終了(末端が欠けないことを保証)
   - まだ進行中なら、`this.elapsed >= this.nextSampleAt` の間ループして1点ずつ記録し、
     そのつど `this.nextSampleAt += SAMPLE_INTERVAL`(大きな dt で複数間隔をまたいでも漏れなく記録)
3. サンプルの `t` は記録時の `this.elapsed`(終了点は実際の経過時刻、最大 `duration`)

これにより:
- 最長 300 秒・間隔 0.5 秒で履歴は最大約 600 点 + 最終点。描画コストは軽微
- 各サンプルの `gu + choki + pa` はその時刻の `aliveCounts()` と常に一致する

## 描画(index.html)

- HTML: メイン canvas の直後にグラフ用 canvas を追加
  - `<canvas id="graph" width="800" height="160"></canvas>`
- レイアウト: `#board` を `flex-direction: column` にし、メイン canvas とグラフ canvas を縦に積む。
  既存の `max-width/max-height` により両方が領域内に収まる。overlay(勝者表示)は従来どおりメイン盤面に重なる
- 新関数 `drawGraph(sim)`(`draw()` と同じ流儀、グローバルの `gctx` を使用):
  - 背景クリア
  - 軸: 左(y)と下(x)に細い軸線。薄いグリッド線(色 `#2c2c54` 程度)を数本
  - 目盛りラベル: y 軸に `0` と `n`(初期人数)、x 軸に `0` と `duration` 秒(フォント小・色 `#888`)。最小限でよい
  - 折れ線: `gu/choki/pa` の3本。`history` を走査し、`t → x`、`count → y` に線形変換して `lineTo` で結ぶ
    - x 変換: `padding + (t / duration) * plotW`
    - y 変換: `padding + (1 - count / n) * plotH`(上が多い)
    - `n === 0` の防御: 初期人数0なら描画スキップ(0除算回避)
  - 凡例は既存サイドバーの人数表示と色が一致しているため、グラフ内には付けない(YAGNI)
- 呼び出し箇所:
  - `loop()`: `draw(sim)` の直後に `drawGraph(sim)`
  - `reset()`: 盤面描画と同様にグラフも初期化描画(開始時点の1点のみ。実質ほぼ平坦)
  - `start()` は `loop()` 経由で描画されるため追加不要

## エラー処理・エッジケース

| ケース | 対処 |
|---|---|
| 初期人数が極端に多い/少ない | y 軸は 0〜n の固定スケール。n=1 でも 0〜1 で描ける |
| `n = 0`(将来スライダー下限が変わった場合) | グラフの折れ線描画をスキップ(0除算回避)。軸のみ描く |
| 大きな dt(タブ復帰など) | `dt` は `MAX_DT` でクランプ済み。さらに while ループで間隔をまたいでも漏れなく記録 |
| 即終了(膠着が開始直後に成立) | 終了点を必ず1点記録するため、開始点+終了点の2点で線が引ける |
| 時間切れちょうど | `timeLeft` は従来どおり 0 にクランプ。終了点の `t` は概ね `duration` |

## テスト(node:test に追加。既存テストは変更なしで通る)

- コンストラクタ直後: `history` が長さ1、`{ t: 0, gu: n, choki: n, pa: n }`
- tick を `SAMPLE_INTERVAL` 未満だけ進めてもサンプルは増えない
- tick を間隔ぶん進めるとサンプルが1点増え、`t` がほぼ `elapsed` と一致
- 大きな dt(複数間隔ぶん)を1回の tick で渡すと、またいだ間隔ぶんのサンプルが記録される
- `finished` になった tick で最終状態が1点記録される(間隔の途中でも末端が入る)
- 各サンプルの `gu + choki + pa` が、その時点の生存総数と整合する
- 注入乱数(`rand`)で捕獲が起きるシナリオを作り、人数が減った点が履歴に現れることを確認
